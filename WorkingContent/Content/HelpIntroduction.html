The AssetIQ Introduction Page

AssetIQ is a system, developed by Crayon, for loading asset data into Flexera's FNMS Asset Management system.
For more information about AssetIQ, please refer to the help pages linked of the Index page

AssetIQ Dataflow

Asset data is uploaded to AssetIQ in spreadsheets (.xslx workbooks) as shown in the following dataflow diagram:

The dataflow diagram depicts AssetIQ's primary use-case, where asset data is first prepared offline, then uploaded and staged to the AssetIQ database.
Once in the database, the uploaded data must be validated to ensure it meets all requirements for being loaded into FNMS.
Only after the data passes validation, may it be processed to load it into FNMS.

The diagram shows the asset data progressing through the folowing workflow steps:

  • Prepare
    		•  A Site-Auditor prepares asset data in a spreadsheet that has been instantiated from a template  that can be downloaded from AssetIQ
  • Upload
    		•  A Site-Auditor uploads a prepared spredasheet using one of the upload pages in AssetIQ
  • Stage
    		•  A Site-Auditor "stages" a previously uploaded spreadsheet, to load its data into the AssetIQ database
  • Validate
    		•  An Site-Auditor runs AssetIQ's data validation processing on staged data
  • Process
    		•  An Asset-Manager runs AssetIQ's FNMS import processing on validated data
  • Manage
    		•  An Asset-Manager "manages" asset data in FNMS
  • Report
    		•  An Asset-Manager runs one of FNMS's asset reports review the asset data in FNMS
